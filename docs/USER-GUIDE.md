## 2. For everyone — fixing a sensor problem

### Ask Dr. Paani 🩺 (the fastest way)

Dr. Paani is the assistant on the home page — *"Sensor acting up? Let's
diagnose it together."*

1. Tap **Ask Dr. Paani**.
2. Type what's wrong **in your own words**. All of these work:
   - "senser not working" *(spelling mistakes are fine)*
   - "pH reading keeps changing"
   - "meter kharab hai"
3. Dr. Paani will either:
   - **Walk you through it step by step** — answer the questions it asks by
     tapping the buttons (Yes / No / Done / That didn't work). Just follow along.
   - **Ask you what the sensor is doing** — tap the option that matches, or type it.
   - **Ask which sensor it is** — tap the type, brand, and model. Not sure? Pick
     the type and tap through; general guidance still works.
   - **Give a short answer with sources** — every answer cites the exact document
     it came from. Tap a source to read the original.

### When Dr. Paani says someone else is needed

Some problems need an electrician, the vendor, or someone authorized to switch
equipment. Dr. Paani will show you **who to contact and how to reach them** —
pick the person listed for *your* plant. You can also tap **Log a support
ticket** so the issue is recorded.

### Did it help?

After each answer, tap **Solved** or **Didn't help**. This genuinely improves
the tool — unanswered questions are reviewed and turned into new guidance.

### Searching directly

Prefer reading documents yourself?
- **Home page search bar** — type a model number, brand, or problem.
- **Browse** — filter every document by type, category, or brand.
- **Sensors** — the full catalog; open any sensor to see everything we have on it.

If nothing matches, Dr. Paani can **search the web** for you — web answers are
clearly marked as *unverified*, so double-check before acting.

## 3. For uploaders — adding documentation

You have an **Upload** button in the top bar.

1. Tap **Upload** and pick the PDF (manual, datasheet, procedure, …).
2. Choose the **document type** and the **sensor** it belongs to
   (brand + model). If the sensor isn't listed, add it right there.
3. Submit. Your file goes to the review team — it is **not** public yet.
4. Track progress under **Uploads** (My submissions):
   - **Pending** — waiting for review
   - **Changes requested** — open it to see the reviewer's note, fix, resubmit
   - **Approved** — merged into the sensor's reference; searchable & used by Dr. Paani
   - **Rejected** — the note explains why
5. You'll get a 🔔 notification when a decision is made.

**Good uploads:** original vendor PDFs, one document per file, correct
brand/model. The clearer the source, the better Dr. Paani's answers.

## 4. For admins — running the tool

Everything lives under **Admin** in the top bar.

### Review queue
New submissions land here. Open one to:
- Read the extracted text (edit it if the PDF scanned badly).
- Check the **AI match banner** — it warns if the document looks like it
  belongs to a different sensor or category.
- Check the **duplicate warning** — it flags near-identical existing content.
- **Approve** into the right activity section (Install, Configure, Clean,
  Calibrate, Troubleshoot & Repair, …), **request changes**, or **reject** —
  the uploader is notified automatically.

### Diagnostic flows *(what powers Dr. Paani's step-by-step walks)*
- Whenever you approve a submission, the AI **automatically drafts** 1–3
  step-by-step diagnostic flows from the new content. A badge shows how many
  drafts await review.
- Open **Admin → Diagnostic flows**, expand a draft, read the tree
  (Ask → Do → Fixed / Escalate), edit any step's wording, then **Approve**.
  Only approved flows ever reach users.
- You can also generate flows from any existing document with
  **Generate with AI**.

### Escalation directory *(who Dr. Paani tells users to call)*
At the bottom of the Diagnostic flows tab:
- Each **skill** (electrical engineer, vendor support, equipment access, …)
  can have several people:
  - **Default** — used everywhere
  - **Only at plant X** — that plant's person
  - **Only for make Y** — that manufacturer's support line
- Fill in the name + phone for each. Keep this current — it's what field
  technicians will see mid-diagnosis.

### Diagnostic routing rules *(per-sensor)*
On each sensor's page: **Generate with AI**, then approve the problem →
procedure rules that look right. Approved rules let Dr. Paani jump straight
to the exact section.

### Insights
Watch **Didn't help** feedback, unanswered questions, and web-search usage —
these tell you what documentation to add next.

### Users, categories, document types
Manage accounts and taxonomy under the corresponding Admin tabs. Roles:
- **Viewer** — search, browse, ask Dr. Paani
- **Uploader** — all of the above + submit documents
- **Admin** — everything, including review and approval
