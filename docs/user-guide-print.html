For everyone

Fixing a sensor problem

Ask Dr. Paani 🩹 — the fastest way

Dr. Paani is the assistant on the home page — “Sensor acting up? Let’s diagnose it together.” Type what’s wrong in your own words. All of these work:

  • “senser not working” (spelling mistakes are fine)
  • “pH reading keeps changing”
  • “meter kharab hai”

Dr. Paani will do one of four things:

  • Walk you through it step by step — answer its questions by tapping the buttons (Yes / No / Done / That didn’t work) and follow along.
  • Ask what the sensor is doing — tap the option that matches, or type it.
  • Ask which sensor it is — tap the type, brand, and model. Not sure? Pick just the type; general guidance still works.
  • Answer with sources — every answer cites the exact document it came from. Tap a source to read the original.

When someone else is needed

Some problems need an electrician, the vendor, or a person authorized to switch equipment. Dr. Paani shows who to contact and how to reach them — pick the person listed for your plant. You can also tap Log a support ticket so the issue is recorded.

After every answer, tap Solved or Didn’t help. This genuinely improves the tool — unanswered questions are reviewed and turned into new guidance.

Reading a sensor reference

Open any sensor reference to read its consolidated guidance. Tap Aa to set a comfortable text size, typeface, and background (white, sepia, or dark) — like an e-reader. A progress bar tracks how far you’ve read.

Searching directly

  • Home search bar — type a model number, brand, or problem.
  • Browse — filter every document by type, category, or brand.
  • Sensors — the full catalog; open any sensor to see everything we have.

If nothing matches, Dr. Paani can search the web for you — web answers are clearly marked unverified, so double-check before acting.

For uploaders

Adding documentation

  1. Tap Upload in the top bar and pick the PDF (manual, datasheet, procedure…).
  2. Choose the document type and the sensor it belongs to (brand + model). If the sensor isn’t listed, add it right there.
  3. Submit. Your file goes to the review team — it is not public yet.
  4. Track progress under Uploads. You’ll get a notification on every decision.
Pending

open it, read the reviewer’s note, fix, resubmit
Approved

Rejected

Good uploads: original vendor PDFs, one document per file, correct brand and model. The clearer the source, the better Dr. Paani’s answers.

For admins

Running the tool

Everything lives under Admin in the top bar.

Review queue

  • Read the extracted text (edit it if the PDF scanned badly).
  • Heed the AI match banner — it warns when a document looks filed under the wrong sensor or category — and the duplicate warning.
  • Approve into the right activity section (Install, Configure, Clean, Calibrate, Troubleshoot & Repair…), request changes, or reject. The uploader is notified automatically.

Diagnostic flows — what powers the step-by-step walks

  • Every approval auto-drafts 1–3 diagnostic flows from the new content; a badge shows drafts awaiting review.
  • Open a draft, read the tree (Ask → Do → Fixed / Escalate), edit any step’s wording, then Approve. Only approved flows reach users.
  • Generate with AI back-fills flows from any existing document.

Escalation directory — who Dr. Paani tells users to call

Tap a skill (electrical engineer, vendor support, equipment access…) to add people: a Default used everywhere, plant-specific people, and make-specific vendor contacts. Keep names and phone numbers current — field technicians see these mid-diagnosis.

Invite teammates

In Users, use Invite a teammate: enter their work email and role (viewer, uploader, or admin). They receive an email link and set their own password on arrival.

Recycle bin — deletions are reversible

Deleting a reference or submission moves it to the Recycle bin, where it stays for 30 days. Restore brings it back complete (search and Dr. Paani included); Delete forever is available inside the bin. After 30 days, items are removed automatically.

Routing rules, insights, and taxonomy

  • On each sensor’s page: Generate with AI, then approve the problem → procedure rules that look right.
  • Insights shows “Didn’t help” feedback, unanswered questions, and web-search usage — your guide to what documentation to add next.
  • Manage accounts, categories, and document types in the matching tabs. Roles: Viewer (search, browse, ask), Uploader (+ submit documents), Admin (everything).
